
# 静态部署说明

若要部署此应用为可公开访问的网站，请按照以下步骤操作:

## 构建应用
1. 运行 `npm run build` 命令构建应用
2. 构建完成后，`dist` 文件夹包含所有静态文件

## 部署选项

### 选项1: GitHub Pages
1. 创建GitHub仓库
2. 推送代码并在设置中启用GitHub Pages
3. 选择 `gh-pages` 分支或 `/docs` 文件夹作为部署源

### 选项2: 阿里云OSS
1. 在阿里云OSS创建Bucket
2. 上传 `dist` 文件夹中的所有文件
3. 设置Bucket为"公共读取"
4. 启用"静态网站托管"功能

### 选项3: 任何静态服务器
1. 将 `dist` 文件夹中的文件上传到任何支持静态网站的服务器
2. 确保index.html能够被正确访问

## 注意事项
- 应用使用相对路径加载资源，确保在服务器配置中设置正确的基础URL
- 如果部署到子目录，可能需要调整 `vite.config.ts` 中的 `base` 配置
